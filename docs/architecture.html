// terraform-aws-sagemaker-mlops

Comprehensive AWS SageMaker MLOps platform with model registry, pipelines, endpoints, feature store, experiments, and model monitoring.

Terraform ≥ 1.5.0 AWS ≥ 5.0 MIT License

Executive Summary

This module provisions a complete end-to-end MLOps platform on AWS SageMaker. It creates a SageMaker Domain with user profiles, model registry for versioning and approval workflows, ML pipelines for orchestration, multi-type inference endpoints (real-time, serverless, async), feature store with online/offline storage, experiment tracking, and model monitoring schedules. An IAM execution role is optionally created with least-privilege policies. All resources are tagged consistently and support VPC-only network isolation.

Production Readiness Checklist

  • Set app_network_access_type = "VpcOnly" for network isolation
  • Enable data capture on real-time endpoints for model monitoring
  • Configure model monitoring schedules with baseline constraints
  • Use separate execution roles per user profile in multi-tenant environments
  • Store pipeline definitions in S3 for version control
  • Enable offline store in Feature Store with S3 for historical data
  • Set up SNS notifications on async endpoints for completion tracking
  • Tag all resources with Environment, Team, and CostCenter
  • Use model registry approval workflows before production deployment
  • Enable CloudWatch alarms for endpoint invocation errors and latency

Deployment

module "sagemaker_mlops" {
  source = "path/to/terraform-aws-sagemaker-mlops"

  name       = "my-ml-project"
  vpc_id     = "vpc-0123456789abcdef0"
  subnet_ids = ["subnet-abc", "subnet-def"]

  user_profiles = {
    data-scientist = {}
  }

  endpoints = {
    inference = {
      type       = "realtime"
      model_name = "my-model"
    }
  }

  tags = {
    Environment = "dev"
  }
}

terraform init
terraform plan
terraform apply
